# 钉钉学习机器人 设计讨论记录

> 目标：做一个服务孩子学习的钉钉机器人——总结与传递信息、作业批改、答疑讲题。
> 本文是方案讨论记录（尚未写代码），用于后续落地参考。

## 1. 先分清两件事

| | 是什么 | 你的场景 |
|---|---|---|
| **A. 做一个"钉钉 bot"** | 钉钉里能收/发消息的机器人 | ✅ 主要是这个 |
| **B. 让 AI 通过 MCP"调用钉钉"** | 把"发钉钉"做成工具给 LLM 调 | 起步不需要 |

结论：核心需求（总结 + 传递 + 批改 + 答疑）**主要是 A**，**起步不需要 MCP**。

## 2. 钉钉 bot 的三种做法

| 方式 | 能力 | 适合 | 公网需求 |
|---|---|---|---|
| ① 群自定义机器人（Webhook） | **只能往群里推**，单向 | 每日学习总结、通知 | 不需要（主动 POST 出去） |
| ② 企业内部应用 + 机器人 | **双向**：收 @消息、回复、卡片 | 想问答/批改 | 回调地址 **或** Stream 模式 |
| ③ **Stream 模式**（推荐做双向） | 同②，后端**出站 WebSocket** 连钉钉 | 后端在家里 Mac（NAT 后） | **不需要公网** |

- 只做"总结+推送" → 用 ①Webhook，最简单。
- 要能收消息/批改/问答 → 用 ③Stream 模式（官方 Python SDK `dingtalk-stream`），出站连接，家里 Mac 完美适配。

## 3. 关于 MCP
- MCP 是"给 LLM 暴露工具"的协议。钉钉本身提供的是 **REST API + 机器人 SDK**，不是 MCP。
- 让 AI "调用钉钉"两条路：
  1. **不用 MCP（推荐起步）**：后端直接调钉钉 API/webhook，"豆包生成总结 → 代码 POST 给钉钉"。
  2. **用 MCP（想让 agent 自主发钉钉时）**：把"发钉钉消息"包成 MCP tool。社区有钉钉 MCP server；官方是否有需查证；也可自己写薄薄一层包装。
- 结论：先"豆包总结 → 推 webhook"，等做成多工具 agent 再 MCP 化。

## 4. Mac 不需要装钉钉软件
- **桌面钉钉 App** = 人看消息用（装手机/电脑，你和家人**看**总结）。
- **bot 后端** = 程序调云端 API（装 Mac，只需**能上网 + Python + 凭证**，出站连接即可）。
- 需要用钉钉 App 的地方（在**手机**上，不是 Mac）：建群、加机器人、拿 webhook、看消息。

## 5. 交互场景：群里有孩子 + 家长

### 关键限制 ⚠️：bot 默认只能看到"@它"的消息
- 钉钉隐私设计：机器人**收不到群里所有聊天**，只收**被 @ 的那条**。
- 交互模型：**孩子 @机器人 + 作业照片/问题** → bot 才处理。
- "读全群消息"需更高的群消息接收权限（隐私敏感）；家庭自用可考虑，但**建议先用 @ 模型**。

### 谁是孩子谁是家长：userId → 角色映射
- bot 收到的每条消息都带发送者 **userId**。
- 后端存映射表：`userId_A→孩子(judy)`、`userId_B→爸爸`…，按角色分权限。

### 作业批改（这条路比 ESP 摄像头更好）
- 流程：孩子 @机器人发作业照片 → bot 下载图片 → **豆包多模态批改** → 回复。
- **复用 `ai_lamp` 已做好的批改能力**（`doubao_grade.py` + 豆包多模态）。
- 优势：**手机拍照画质远好于 ESP 那颗摄像头**，正好绕开"OV2640 画质糊、批改不准"的最大风险。手机 = 作业扫描仪。

### 讲题/答疑
- 孩子 @机器人 + 问题（文字或拍照）→ 豆包 → 回复。
- 用 `ai_lamp` 的**苏格拉底式 tutor 提示词**（先引导不直接给答案），Markdown 卡片回复更清晰。

## 6. 整体架构（复用现有）
```
孩子 @机器人(作业/问题) ─┐
家长 @机器人(总结/设置) ─┤→ Mac后端(钉钉bot, Stream) →(按发送者判角色)
                          │      → 豆包多模态(批改/讲题/总结) → 回复卡片到群
共享"孩子学习记录" ───────┘
```
- 可与陪读精灵 server 放一起（同在 Mac、同用豆包、共享记录）。

## 7. 前置条件
- 建一个团队/企业（家庭团队版免费），建**内部应用**、开机器人 + Stream 模式，拿 AppKey/AppSecret。
- 孩子要有钉钉账号并加入团队和群。
- 申请**图片下载**相关权限（批改要拿到照片）。

## 8. 与 ESP 陪读精灵的分工（互补）
- **钉钉 bot**：异步、有屏幕 + 手机摄像头 → 作业批改、讲题、家长总结。
- **ESP 精灵**：实时语音、无屏 → 陪聊、听写、口头答疑。
- 共享同一份"孩子学习记录"，互相喂数据。

## 9. 待定决策（落地前先敲定）
1. **消息范围**：先"@机器人才响应"（推荐），还是要"读全群消息"（需高权限 + 隐私考量）？
2. **角色与权限**：孩子能用哪些功能、家长能用哪些？
3. **先做哪个能力**：作业批改 / 讲题 / 家长总结——先打通一个。

## 10. 隐私与安全
- 群机器人用**加签**校验（webhook 场景）。
- 孩子数据最小化推送、群成员可控；总结避免放敏感原始内容。
- 全部在家庭团队内闭环。
